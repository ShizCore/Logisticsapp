# SQL-реализация Repository Pattern

Данная ветка содержит реализацию паттерна Repository через сырые SQL-запросы с использованием библиотеки `psycopg2`.

## Особенности реализации

### Файл: `server/repositories/sql_repo.py`

Класс `SqlRawShipmentRepository` реализует интерфейс `IShipmentRepository` через прямые SQL-запросы:

- **Защита от SQL-инъекций**: Все запросы используют параметризованные placeholder'ы (`%s`)
- **Динамическая фильтрация**: Метод `get_filtered()` строит WHERE-условия программно
- **Управление транзакциями**: Явные `commit()` и `rollback()` при ошибках
- **Освобождение ресурсов**: Подключения закрываются в блоке `finally`

## Сравнение с ORM-реализацией

| Критерий | ORM (SQLAlchemy) | Raw SQL (psycopg2) |
|----------|------------------|-------------------|
| Производительность | Средняя (накладные расходы ORM) | Высокая (прямые запросы) |
| Гибкость | Ограничена возможностями ORM | Полная (любой SQL) |
| Читаемость | Высокая (Python-код) | Средняя (SQL в строках) |
| Безопасность | Автоматическая | Ручная (параметризация) |
| Поддержка | Проще (меньше кода) | Сложнее (больше кода) |

## Пример запроса

```python
query = """
    SELECT s.shipment_id, s.client_id, s.weight_kg, s.status
    FROM shipments s
    WHERE s.status = %s AND s.weight_kg >= %s
"""
